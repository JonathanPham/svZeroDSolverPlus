\DoxyCodeLine{28 \textcolor{comment}{ * \# Background}}
\DoxyCodeLine{29 \textcolor{comment}{ *}}
\DoxyCodeLine{30 \textcolor{comment}{ * Zero-\/dimensional (0D) models}}
\DoxyCodeLine{31 \textcolor{comment}{ * are lightweight methods to simulate bulk hemodynamic quantities in the}}
\DoxyCodeLine{32 \textcolor{comment}{ * cardiovascular system. Unlike 3D and 1D models, 0D models are purely time-\/dependent;}}
\DoxyCodeLine{33 \textcolor{comment}{ * they are unable to simulate spatial patterns in the hemodynamics.}}
\DoxyCodeLine{34 \textcolor{comment}{ * 0D models are analogous to electrical circuits. The flow rate simulated by}}
\DoxyCodeLine{35 \textcolor{comment}{ * 0D models represents electrical current, while the pressure represents voltage.}}
\DoxyCodeLine{36 \textcolor{comment}{ * Three primary building blocks of 0D models are resistors, capacitors, and inductors}}
\DoxyCodeLine{37 \textcolor{comment}{ * Resistance captures the viscous effects of blood flow, capacitance represents}}
\DoxyCodeLine{38 \textcolor{comment}{ * the compliance and distensibility of the vessel wall, and inductance represents}}
\DoxyCodeLine{39 \textcolor{comment}{ * the inertia of the blood flow. Different combinations of these building blocks,}}
\DoxyCodeLine{40 \textcolor{comment}{ * as well as others, can be formed to reflect the hemodynamics and physiology of}}
\DoxyCodeLine{41 \textcolor{comment}{ * different cardiovascular anatomies.These 0D models are governed}}
\DoxyCodeLine{42 \textcolor{comment}{ * by differential algebraic equations (DAEs).}}
\DoxyCodeLine{43 \textcolor{comment}{ *}}

\DoxyCodeLine{46 \textcolor{comment}{ * \#\# Model}}
\DoxyCodeLine{47 \textcolor{comment}{ *}}
\DoxyCodeLine{48 \textcolor{comment}{ * The solver uses a highly modular framework to model the vascular anatomy,}}
\DoxyCodeLine{49 \textcolor{comment}{ * using individual classes to represent different 0D elements of the}}
\DoxyCodeLine{50 \textcolor{comment}{ * model. The elements are part of the MODEL namespace. Currently}}
\DoxyCodeLine{51 \textcolor{comment}{ * supported elements are:}}
\DoxyCodeLine{52 \textcolor{comment}{ *}}
\DoxyCodeLine{53 \textcolor{comment}{ * * MODEL::BloodVessel: RCL blood vessel respresentation with optional stenosis}}
\DoxyCodeLine{54 \textcolor{comment}{ * * MODEL::Junction: Junction element with arbitrary inlets and outlets.}}
\DoxyCodeLine{55 \textcolor{comment}{ * * MODEL::FlowReference: Prescribed flow boundary condition}}
\DoxyCodeLine{56 \textcolor{comment}{ * * MODEL::RCRBlockWithDistalPressure: RCR Windkessel boundary condition}}
\DoxyCodeLine{57 \textcolor{comment}{ *}}
\DoxyCodeLine{58 \textcolor{comment}{ * More information about the elements can be found on their}}
\DoxyCodeLine{59 \textcolor{comment}{ * respective pages. The elements are connected to each other via nodes}}
\DoxyCodeLine{60 \textcolor{comment}{ * (see MODEL::Node). Each node corresponds to a flow and a pressure value}}
\DoxyCodeLine{61 \textcolor{comment}{ * of the model. The MODEL::DOFHandler handles the degrees-\/of-\/freedom (DOF) of the}}
\DoxyCodeLine{62 \textcolor{comment}{ * system by assigning DOF indices to each element that determine the location}}
\DoxyCodeLine{63 \textcolor{comment}{ * of the local element contribution in the global system. The complete model}}
\DoxyCodeLine{64 \textcolor{comment}{ * is stored and managed by the MODEL::Model class.}}
\DoxyCodeLine{65 \textcolor{comment}{ *}}
\DoxyCodeLine{66 \textcolor{comment}{ * \#\# Algebra}}
